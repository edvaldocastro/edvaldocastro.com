[Troubleshooting] - Erro ao habilitar .Net Framework 3.5 no Windows Server 2012 R2

14 Nov 2014 · Database Administration, Tech, Virtual PASS BR · .net framework 3.5, .netframework3.5, english, erro, erroclusterSQL, SERVERsqlserver2014, sqlpass, troubleshooting, VirtualPassBr, VirtualPassPT

Demanda:

Um dos clientes da Sr. Nimbus, solicitou a instalação de um cluster Windows Server 2012 R2 com SQL Server 2014.

Um dos requisitos do SQL Server 2012 / 2014 é a habilitação do ".Net Framework 3.5" que pode ser realizado tanto pela interface gráfica quanto via linha de comando, utilizando por exemplo o "Dism.exe"

Para habilitar o .Net Framework 3.5 via linha de comando, basicamente deve-se especificar a habilitação da feature e o caminho dos arquivos de onde serão instalados, conforme exemplo a seguir:

Dism.exe /online /enable-feature /featurename:NetFx3 /All /Source:D:sourcessxs

Problema:

O último trecho ( /Source:D:sourcessxs) da linha de comando supracitada, é onde deve-se informar a fonte para a habilitação do .Net Framework 3.5.

O erro retornado é que não foi possível fazer o download (da origem informada) dos arquivos para a necessários para esta habilitação, mesmo o caminho tendo sido informado corretamente.

“...the source files could not be downloaded...”

Este erro acontece devido a atualizações de segurança (KB2966828 e KB2966827) do .Net Framework 3.5 que podem ter sido instaladas antes de sua habilitação, automaticamente via windows update (caso o servidor tenha acesso à internet), via WSUS ou mesmo manualmente.

Solução:

  •  Run: Appwiz.cpl (Adicionar ou Remover programas)
  • Clique em Installed Updates
  • Na caixa de pesquisa, procure por KB2966828 e KB2966827
  • Desinstale ambos, ou apenas o que encontrar
  • Faça o procedimento de habilitação do .NetFramework 3.5
  • Faça o download
    • KB2966828
      • http://www.microsoft.com/en-us/download/details.aspx?id=43926
    • KB2966827
      • http://www.microsoft.com/en-us/download/details.aspx?id=43951
  • Reinstale o(s) KB(s) removido(s).
  • Prossiga normalmente com a instalação do SQL Server.
